## 1. 서비스/업무 목표
- 서산시 유휴 공유재산 정보를 지도에서 빠르게 조회할 수 있게 한다.
- 공무원/담당자가 개발 지식 없이도 엑셀 업로드로 데이터를 최신화할 수 있게 한다.
- 주소 기반 데이터(정적 목록)를 필지 경계(geometry) 기반 데이터(시각화 가능)로 변환해 활용도를 높인다.
- 대민 공개 정보 접근성을 높이되, 관리자 기능은 내부망·인증 기반으로 분리 운영한다.

## 2. 사용자 경험 목표
- 일반 사용자에게는 검색/필터/목록/지도 상호작용이 즉시 반응하도록 제공한다.
- 지도 화면에서 조건 검색, 목록 탐색, 상세 팝업 확인, 전체 목록 다운로드를 일관된 흐름으로 제공한다.
- 관리자에게는 로그인 후 업로드/설정/비밀번호 변경/통계를 단일 콘솔에서 처리하게 한다.
- 장애나 오류 발생 시 사용자에게 원인을 이해할 수 있는 메시지를 제공한다.

## 3. 데이터 정확성 목표
- 업로드 엑셀의 필수 컬럼 누락, 행 단위 값 오류, 행 수 초과를 서버에서 강제 검증한다.
- 저장 전 정규화 과정을 통해 형식 불일치 데이터를 줄인다.
- 공개 API 응답에는 내부용/민감 필드를 노출하지 않고 필요한 필드만 제공한다.
- 지오메트리 미확보 건수를 추적하고 보강 작업 결과(성공/실패)를 관리한다.

## 4. 보안 목표
- 관리자 페이지는 내부 허용 IP 대역에서만 접근 가능하게 한다.
- 세션 기반 인증과 CSRF 토큰 검증으로 관리자 POST 요청을 보호한다.
- 로그인 실패 누적 시도 제한으로 계정 추측 공격을 완화한다.
- 보안 헤더(CSP, X-Frame-Options, X-Content-Type-Options)를 기본 적용한다.
- 설정 변경/비밀번호 변경은 인증된 관리자만 수행하게 한다.

## 5. 운영성/유지보수 목표
- 라우터-서비스-리포지토리-클라이언트 계층 분리를 유지해 변경 범위를 국소화한다.
- 환경변수 중심 설정으로 배포 환경별 제어를 단순화한다.
- 요청 ID와 구조화 로그로 장애 원인 추적성을 확보한다.
- 테스트(단위/통합/API/보안 회귀)로 핵심 회귀를 조기 탐지한다.
- 유지보수 문서와 체크리스트로 운영 절차를 표준화한다.
- 설정 변경은 운영 절차(재시작/검증)와 함께 관리해 반영 시점 혼선을 줄인다.

## 6. 외부 연동/성능 목표
- VWorld API 의존 구간에 타임아웃/재시도/백오프를 적용해 일시 장애에 견고하게 동작한다.
- 공개 지도 데이터 조회는 페이지네이션으로 응답 크기를 제어한다.
- 대용량 업로드 시 파일 크기/행 수 제한으로 서버 과부하를 예방한다.
- 지오메트리 보강을 비동기 백그라운드 처리해 업로드 응답 지연을 줄인다.

## 7. 관측/의사결정 지원 목표
- 검색/클릭/웹 방문 이벤트를 수집해 실제 사용 패턴을 파악한다.
- 관리자 통계 화면에서 핵심 지표(검색수, 클릭수, 방문자, 체류시간, 추세)를 제공한다.
- 원시 질의 로그 내보내기로 정책/기능 개선 근거 데이터를 확보한다.

## 8. 품질 목표(테스트 및 릴리스)
- 최소 품질선으로 `pytest -q` 통과와 커버리지 기준(설정값 기준)을 유지한다.
- 타입 체크, 빌드 검증, 컴파일 검증을 배포 전 필수 절차로 유지한다.
- 보안/접근 제어/CSRF/프록시 신뢰 경계 관련 회귀 테스트를 유지한다.

## 9. 비목표(현재 범위 밖)
- 다중 관리자 계정/권한(Role) 체계 제공은 현재 범위 밖이다.
- 실시간 분산 작업큐 기반 장기 잡 오케스트레이션은 현재 범위 밖이다.
- 외부 사용자 인증/개인화 기능은 현재 범위 밖이다.
- 고급 BI 대시보드(세그먼트/퍼널/코호트) 수준 분석은 현재 범위 밖이다.

## 10. 향후 목표 후보(로드맵)
- 지오메트리 보강 작업의 내구성 개선(작업큐/재시도 정책 고도화).
- 설정 변경의 무중단 반영 또는 안전한 재시작 자동화.
- 공개 다운로드 파일 제공 방식의 대용량/고동시성 최적화.
- 운영 메트릭 표준화(오류율, 외부 API 실패율, 잡 지연, 업로드 실패율).
- 이벤트/원시 로그 데이터 보존 및 정리 자동화 정책 확립.
