## 문서 운영 원칙
- 코딩 원칙/스타일 변경은 `engineering-guidelines.md`를 먼저 갱신한다.
- 목표/구조/운영/보안 문서는 본인 역할 범위만 유지하고 중복 규칙을 복제하지 않는다.
- 기능 변경 시 해당 문서(구조/운영/보안)를 동시 갱신한다.
- 주요 릴리스 전 문서 링크와 내용 최신성을 점검한다.
